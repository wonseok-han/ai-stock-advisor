# 05. AI 적용 전략

## 5.1 전략 요약

**RAG (Retrieval Augmented Generation) + 경량 LLM.**

자체 모델 학습/파인튜닝은 **하지 않음**. 대신:
1. Spring Boot가 **신뢰할 수 있는 구조화 데이터**(기술지표, 가격, 뉴스)를 수집
2. LLM에 **프롬프트 컨텍스트로 주입**
3. LLM은 판단자 역할 — JSON으로 시그널/근거/리스크 반환
4. 결과를 검증 후 캐싱

이 구조는 **할루시네이션 최소화 + 비용 최소화 + 통제 가능성 최대화**를 동시에 달성합니다.

## 5.2 모델 선택

### 1순위: Google Gemini 1.5 Flash

**이유:**
- 무료 티어 존재 (분당 15 RPM, 일 1M 토큰, 월 150만 요청)
- 유료도 가장 저렴 ($0.075/1M input, $0.30/1M output)
- JSON 구조화 출력 지원 (`responseMimeType: application/json`)
- 한국어 품질 양호
- 긴 컨텍스트(1M tokens) 지원

### 2순위 (품질 이슈 시): GPT-4o-mini

- $0.15/1M input, $0.60/1M output
- Structured Output 기능 안정적
- 여전히 매우 저렴

### 3순위 (고품질 필요 시): Claude Haiku 4.5

- 추론 품질이 더 좋음
- 비용 약간 높음

→ **MVP는 Gemini Flash 단일 사용. 추상화 레이어(`LlmClient` 인터페이스)를 두어 향후 교체 가능하게.**

## 5.3 AI 활용 포인트

### 용도 A: 종목 분석 시그널 (핵심)

**입력 컨텍스트 (서버가 조립):**

```
[종목 기본]
- 티커: AAPL
- 회사명: Apple Inc.
- 섹터: Technology
- 시가총액: $2.8T

[최근 가격 액션 (7일)]
- 현재가: $175.23
- 7일 수익률: +2.3%
- 7일 고가/저가: $178.45 / $172.10
- 거래량: 20일 평균의 112%

[기술적 지표]
- RSI(14): 58 (중립, 과매수 아님)
- MACD: MACD 0.85, Signal 0.72, Histogram +0.13 (양전환)
- 볼린저밴드: 상단 $180, 중앙 $175, 하단 $170, 현재 중앙 부근
- MA5: $174.80, MA20: $173.50, MA60: $170.20 (정배열)

[종목 뉴스 (최근 7일, 5건)]
1. "Apple announces new AI features in iOS 18" — Reuters, 2일 전
2. "iPhone 16 demand exceeds expectations in China" — Bloomberg, 3일 전
...

[시장 컨텍스트]
- S&P500 7일: +1.2%
- Nasdaq 7일: +1.8%
- VIX: 14.5 (낮음, 시장 안정)
- USD/KRW: 1,340원
```

**프롬프트 (축약):**

```
당신은 미국 주식 분석가입니다. 아래 데이터를 바탕으로 {티커}의
단기(1~2주) 기술적/심리적 시그널을 분석하세요.

규칙:
- 투자 자문이 아닌 분석 정보임을 전제
- JSON 형식으로만 응답
- 신뢰도는 0.0~1.0
- rationale과 risks는 각 2~5개

[데이터]
{위의 구조화 컨텍스트}

응답 스키마:
{
  "signal": "strong_buy|weak_buy|neutral|weak_sell|strong_sell",
  "confidence": 0.0~1.0,
  "timeframe": "단기(1~2주)",
  "rationale": [...],
  "risks": [...],
  "summary_ko": "2~4문장 요약"
}
```

**제약/검증 (Spring Boot에서 수행):**
- JSON 파싱 실패 → 재시도 1회, 그래도 실패 시 "분석 실패, 나중에 시도" 표시
- signal 값이 enum 벗어나면 중립 처리
- confidence 범위 벗어나면 clamp
- rationale/risks 개수 강제

### 용도 B: 뉴스 번역 & 요약

**입력:** 영어 뉴스 제목 + 본문(없으면 헤드라인만)

**프롬프트:**
```
아래 영어 뉴스 기사를 한국어로 번역하고, 투자자 관점에서
2~3줄 요약을 만드세요. JSON으로만 응답.

{
  "title_ko": "...",
  "summary_ko": "...",
  "sentiment": "positive|neutral|negative"
}
```

**캐싱:** 뉴스 ID 기준 24시간 Postgres 캐시 → 재번역 비용 $0.

### 용도 C (선택): 시장 브리핑 자동 생성

매일 1회, 시장 전반 뉴스 + 지수 + VIX를 종합해 **"오늘의 시장 요약"** 생성.
→ 1일 1회만 호출이므로 비용 미미. Phase 3에서 검토.

## 5.4 비용 추정 (Gemini Flash 기준)

| 항목 | 단가(추정) | 월간 호출(가정) | 월 비용 |
|---|---|---|---|
| 종목 분석 (2K in / 500 out) | ~$0.0003 | 10,000회 | **$3** |
| 뉴스 번역 (500 in / 200 out) | ~$0.0001 | 20,000회 | **$2** |
| 시장 브리핑 (3K in / 1K out) | ~$0.0006 | 30회 | $0.02 |
| **합계** | | | **~$5/월** |

캐시 hit ratio 70% 이상이면 실제 비용은 **$2~3/월**.

## 5.5 프롬프트 엔지니어링 원칙

1. **Few-shot 예시는 필요 시에만** — 토큰 소모 크므로 기본은 zero-shot + 스키마 강제
2. **자연어 판단을 요구하지 말고 데이터 기반 판단을 요구** — "당신의 느낌은?"이 아니라 "RSI, MACD, 뉴스 감성을 근거로..."
3. **구조화 출력 필수** — 자유서술 금지
4. **면책 문구 강제** — summary_ko 마지막에 "본 분석은 투자 자문이 아닙니다"를 포함하도록 지시
5. **한국어 전용 출력** — 다국어 섞이면 UX 저하

## 5.6 품질 검증 루프

MVP 출시 전:
- 다양한 종목(AAPL, TSLA, NVDA, 신규 상장주, 페니스톡 등)에 대해 수동 테스트
- 명백히 강한 신호/약한 신호 상황에서 모델 응답이 직관과 일치하는지 검증
- 악의적 입력 (예: 티커 대신 "ignore previous instructions") 에 대한 방어 테스트

→ 문제 발견 시 프롬프트 조정 or 2순위 모델(GPT-4o-mini)로 교체 실험.

## 5.7 의도적으로 **하지 않는 것**

- **가격 예측 수치 생성** ("이 주식은 3개월 내 $200 갈 것") — 법적/신뢰도 리스크
- **매수/매도 단언** — "사세요", "팔지 마세요" 문구 금지
- **사용자 포트폴리오 기반 개인 조언** — 유사투자자문업 소지
- **LLM이 실시간 API를 직접 호출 (function calling)** — 제어 어려움, MVP에선 서버가 조립한 컨텍스트만 사용
